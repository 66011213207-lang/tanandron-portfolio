# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

This is a plain static site ŌĆö no bundler, no package.json, no test/lint scripts.

**Preview locally.** `index.html` loads `js/main.js` as an ES module (`<script type="module">`),
so it must be served over http(s) ŌĆö opening the file directly (`file://`) will fail with a CORS
error. Serve from the repo root, e.g.:
```
npx serve .
```
then open the printed `localhost` URL.

**Regenerate the gallery** after adding/removing photos in `assets/gallery/`:
```
node scripts/build-gallery.js
```
(or double-click `update-gallery.bat` on Windows). This is the only script in the repo and has
no dependencies beyond Node's `fs`/`path`.

**Deploy.** The site is published via GitHub Pages, serving straight from `main` / root ŌĆö pushing
to `main` is the entire deploy step, live in about a minute.

## Architecture

Three files drive the whole site: `index.html`, `css/style.css`, `js/main.js`. No framework, no
build step ŌĆö `js/main.js` renders sections by writing template-string HTML into container
elements and re-running the relevant `render*()` function on state change.

**Design tokens.** All colors/spacing/radii are CSS custom properties defined once in
`:root` in `css/style.css` (navy base + a single yellow accent). `design.md` is the source of
truth for the token table and component specs (button/tab/card states, contrast rule for text on
the yellow accent, motion timings) ŌĆö check it before introducing new colors or components rather
than hardcoding hex values.

**`js/main.js` is organized into comment-delimited blocks, each independent:**
- `PROJECTS` ŌĆö drives the Three.js viewer, the breakdown panel, and the Projects grid. Each entry
  has a `model` (glb/gltf URL), `stats`, and (once real assets exist) a `renders` map ŌĆö see
  `assets/models/README.md` and `assets/renders/README.md` for the expected file-naming
  convention and how to wire a project's `renders` object once real breakdown images/models
  exist.
- `GALLERY_ITEMS` ŌĆö between `/* GALLERY_ITEMS:START */` and `/* GALLERY_ITEMS:END */` markers.
  **Auto-generated by `scripts/build-gallery.js` ŌĆö never hand-edit this array.** The script scans
  `assets/gallery/<Category>/` (each subfolder is one filter-chip category shown on the site),
  turns filenames into captions, and rewrites the block in place. `LOW_PRIORITY_CATEGORIES` in
  that script (currently `Activity`, `Other`) controls which categories sort last under the "All"
  filter instead of alphabetically.
- Three.js viewer setup (scene/camera/renderer/lights/OrbitControls) ŌĆö `three` is loaded from a
  CDN via the import map in `index.html`, not from `node_modules`. `loadModel()` swaps the active
  GLTF; wireframe toggling mutates `.wireframe` on every material in `currentMeshMaterials`
  rather than swapping materials.
- Project picker + breakdown tabs + stats, gallery + filter + lightbox, nav highlighting
  (`IntersectionObserver` on `main .section`), and scroll-reveal (`IntersectionObserver` adding
  `.in-view` to every direct child of `.section`) are each self-contained render functions wired
  up at the bottom of the file under `INIT`.

**Placeholder policy.** No final renders/models are supplied yet for the breakdown panel
(`design.md` ┬¦8) ŌĆö it shows styled placeholder blocks until real assets are dropped into
`assets/models/` / `assets/renders/` and wired into a `PROJECTS` entry.

**`assets/profile/portrait.jpg`** is the About-section photo; its crop is tuned via
`object-position` on `.about-portrait img` in `css/style.css` to keep the subject's eye-line on
the upper rule-of-thirds line ŌĆö recompute rather than eyeballing a new value if the photo changes.

**`.gitignore`** excludes `.claude/`, `.agents/`, and `skills-lock.json` (local Claude Code
tooling state, not part of the site).
